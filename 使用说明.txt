个人工作面板 v1.2.0 使用说明
================================

一、直接启动
双击“个人工作面板.exe”即可。
正式发布版采用 Windows GUI 模式，不会打开 CMD/终端，也不依赖终端持续运行。

二、窗口与托盘
1. 程序默认不显示任务栏图标，只在系统托盘中驻留。
2. Ctrl + Shift + Space：呼出或隐藏工作面板。
3. 托盘图标左键：显示或隐藏窗口。
4. 托盘图标右键：“打开”或“退出”。
5. 顶部“—”：只隐藏到系统托盘，程序继续运行。
6. 顶部“X”：等待本地数据保存后彻底退出应用。

三、项目重命名
1. 在“项目空间”的项目卡片中，点击项目名称即可编辑。
2. 进入项目详情后，也可以点击页面顶部的项目名称编辑。
3. Enter 或点击输入框外：保存名称。
4. Esc：取消本次修改。
5. 空名称不会保存；已完成项目遵循只读规则，不能重命名。

四、小窗模式
1. 小窗默认尺寸为 620×720，最小尺寸为 560×640。
2. 尺寸使用 Windows 逻辑像素，可适配系统 DPI 缩放。
3. To-do 内容会完整换行显示，不会用省略号截断。
4. 仍可拖动窗口边缘按个人需要继续调整。

五、界面动效
1. 页面切换、项目卡片、To-do、弹窗和按钮现在带有短促的反馈动画。
2. 完成 To-do 或结束今日工作时会显示明确但克制的完成反馈。
3. 动画不会延迟数据保存和功能操作。
4. 如果 Windows 开启“减少动画/动画效果”辅助选项，软件会自动将动效缩短至近乎即时。

六、开发模式说明
“npm run tauri -- dev”是开发命令，必须依赖终端显示日志；关闭开发终端会结束开发进程，这是正常行为。
平时使用请直接双击本目录中的 EXE，不要运行开发命令。

七、数据与 Markdown 导出
数据保存在 Windows 当前用户的应用数据目录中。可在设置中启用开机启动，并全选或指定项目导出 Markdown。默认导出到 EXE 所在目录的 export 文件夹；也可以点击“自定义”选择其他文件夹。已完成待办在导出文件中使用 √ 标记。

八、Windows 安全提示
当前 EXE 未购买代码签名证书，首次运行可能触发 SmartScreen。
如确认文件来自本项目，可选择“更多信息”→“仍要运行”。
