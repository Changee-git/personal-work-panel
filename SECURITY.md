# 安全策略

## 支持版本

当前仅维护最新正式版本。

| 版本 | 安全更新 |
| --- | --- |
| 1.1.x | ✅ |
| 1.0.x | ❌ |

## 报告漏洞

请优先通过 GitHub 仓库的 **Security → Report a vulnerability** 私密报告功能提交问题。报告中建议包含：

- 受影响版本与 Windows 版本
- 可复现步骤
- 影响范围
- 建议修复方式（如有）

请勿在公开 Issue 中粘贴密钥、真实工作数据、用户目录、系统日志或可直接利用的漏洞细节。如果仓库尚未启用私密漏洞报告，可先创建一个不含敏感细节的公开 Issue，请求维护者开启私密沟通渠道。

## 安全边界

- 应用本地运行，不提供网络服务端口或远程账号体系。
- 所有用户输入仍视为不可信数据；导出内容不会作为命令执行。
- Release EXE 当前未进行商业代码签名，用户应只从本项目 GitHub Releases 下载并核对发布版本。
