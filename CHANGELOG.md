# 变更记录

暂无。

## [1.2.0] - 2026-07-16

### 新增

- 设置页新增 Markdown 批量导出工作区，支持全选或指定项目、默认路径展示、自定义文件夹及导出反馈。
- 默认导出文件夹为程序所在目录下的 export，并允许用户切换到自定义目录。

### 变更

- 设置页“数据备份”入口改为“导出 Markdown”，不再保留无必要的手动备份入口。
- Markdown 中已完成待办由 [x] 改为 [√]，更直观地表达完成状态。
- 导出交互增加项目勾选、路径切换和成功反馈动画，并遵守 prefers-reduced-motion。
- 清理重复发布程序、开发安装器、构建缓存及独立前端测试文件和测试专用依赖。
- 根目录仅保留一个可直接运行的正式版 EXE，版本号同步升级为 1.2.0（npm、Cargo 与 Tauri）。

### 修复

- 修复根目录旧版 EXE 未更新、旧发布程序后台运行导致新版无法启动的问题。
- 确认清理过程不修改用户 AppData 数据，已有项目、Todo 和 Issue 保持不变。

## [1.1.0] - 2026-07-15

### 新增

- 增加点击项目名称原地重命名，支持 Enter/失焦保存、Esc 取消和空名称保护。
- 增加顶部“退出应用”按钮，并在退出前等待本地保存队列完成。
- 增加独立组件动效层，覆盖视图、列表、卡片、弹窗、导航、按钮和完成状态反馈。
- 增加 `prefers-reduced-motion` 降级，启用 Windows“减少动画”时自动缩短动效。
- 增加项目重命名组件测试和 Windows PE 子系统自动验证脚本。
- 增加 README 的项目总览、项目详情和今日待办截图；截图仅使用虚构演示数据。
- 增加 MIT 许可证、隐私说明和安全报告策略，完成首次公开发布准备。

### 修复

- Windows Release EXE 改为 GUI 子系统，双击启动不再创建 CMD/控制台窗口。
- 保留 Debug 构建的控制台，避免影响开发日志和故障排查。
- 小窗尺寸从物理像素改为逻辑像素，修复高 DPI Windows 环境下窗口过窄的问题。
- To-do 内容在小窗中完整换行显示，不再因可用宽度不足影响阅读。

### 变更

- 统一动效为 120–320ms 三档时长，并采用适合进入、切换和强调反馈的缓动曲线。
- 动效使用 CSS/WebView2 原生能力实现，不增加运行依赖或数据流复杂度。
- 小窗默认尺寸调整为 `620×720` 逻辑像素，最小尺寸调整为 `560×640`。
- 顶部窗口操作明确区分“隐藏到系统托盘”和“退出应用”。
- 清理文档中的开发机绝对路径，发布仓库仅保留可复现构建所需内容。
- 版本号同步更新为 `1.1.0`（npm、Cargo 与 Tauri）。

### 安全

- 扩充 `.gitignore`，排除本地环境文件、安装工具、构建目录、日志、调试符号和发布二进制。
- 发布前增加密钥、个人路径、邮箱、手机号与误纳入二进制检查。
- 仓库不包含真实工作数据、粘贴图片、备份文件或浏览器截图数据源。

## [1.0.0] - 2026-07-14

### 新增

- 完成 Tauri 2 + React 个人工作面板 MVP。
- 支持今日待办、灵感、项目空间、Todo、Issue 和项目总结。
- 支持系统托盘、全局快捷键、任务栏隐藏与主/小窗模式。
- 支持本地 JSON 持久化、图片保存、备份、Markdown 导出和开机启动。

### 修复

- 清除 Tauri/TypeScript JSON 配置中的 UTF-8 BOM，确保工具链可解析。
- 为 Tauri 启用 `protocol-asset` 特性，与资产协议配置保持一致。
